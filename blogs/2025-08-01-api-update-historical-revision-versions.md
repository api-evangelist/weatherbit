<h3 id="historical-data-versioning">Historical Data Versioning</h3><img alt="API Update: Historical Revision Versions" src="https://blog.weatherbit.io/content/images/2025/08/Version-control-Best-Practices-Blog-Hero.png.webp" /><p>We have added a new field to most historical data API&apos;s to indicate revision versions. </p><p><code>&quot;revision_version&quot;</code></p><p>To ensure our service always returns the best possible data for every location - on <strong><em>rare</em></strong> occasions - it is necessary to make changes to either calculated fields, or the underlying data (even when in a &quot;final&quot; state). In this case the <code>&quot;revision_version&quot;</code> field will indicate an updated version number. </p><p><code>revision_version:</code> &quot;MAJOR_VERSION.MINOR_VERSION&quot; - Example: &quot;1.0&quot;</p><p>The minor version will update when:</p><ul><li>Improvements have been made to calculations of derived fields. </li><li>Bug fixes or minor improvements have been made that may only impact specific locations or time frames.</li></ul><p>The major version will update when:</p><ul><li>There are new historical dataset(s) added. </li><li>Major updates to station/data source selection have been made. </li><li>Changes or fixes have been made that impact all data returned by the API.</li></ul><p>As a general practice we recommend querying again for past data to obtain these revisions using both  <strong><code>&quot;revision_status&quot;</code></strong> field, <strong>AND</strong> <strong><code>&quot;revision_version&quot;</code></strong> to determine whether or not to replace data in any locally hosted database. </p><p>More information on this can be found on the following page: <a href="https://help.weatherbit.io/why-does-the-historical-data-change/?ref=blog.weatherbit.io">https://help.weatherbit.io/why-does-the-historical-data-change/</a></p><h3 id="historical-revision-version-api">Historical Revision Version API</h3><p>To make it easier to detect when a major or minor data revision has occurred, we have added an <a href="https://www.weatherbit.io/api/revision-version?ref=blog.weatherbit.io" rel="noreferrer">API</a> that returns this version. As such there is no need to query a historical API directly. </p><p><code>https://api.weatherbit.io/v2.0/history/revision_version?key=YOUR_API_KEY</code>  </p><figure class="kg-card kg-code-card"><pre><code class="language-json">{
  &quot;revision_version&quot;: &quot;1.0&quot;
}</code></pre><figcaption><p><span style="white-space: pre-wrap;">This new API returns a simple JSON response with the current revision version</span></p></figcaption></figure><h2 id="conclusion">Conclusion</h2><p>We hope that our users find value in these updates. We are always looking for ways to improve our API. Feel free to reach out to support@weatherbit.io with any questions or concerns.</p>
